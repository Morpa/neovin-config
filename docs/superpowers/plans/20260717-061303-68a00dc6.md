# Tirar o Neovide + adicionar plugins do cpow/neovim-for-newbs

## Context

A config hoje é pensada pra rodar no **Neovide** (GUI). Quase toda a camada de
atalhos "estilo Zed" (Cmd+P, Cmd+S, Cmd+/, Cmd+B, Cmd+1..9, mover linha, zoom
etc.) vive em `nvim/lua/config/neovide.lua` dentro de um guard
`if not vim.g.neovide then return end` — ou seja, **só carrega dentro do
Neovide**. O objetivo é abandonar o Neovide e rodar no terminal (o usuário usa
**Ghostty**, com `~/.config/ghostty/config` já configurado, Neovim **0.12.4**).

Como o Neovim 0.12 fala o **protocolo de teclado do Kitty** e o Ghostty também,
as teclas Cmd (`<D-...>`) **chegam ao Neovim no terminal** — exceto as
combinações que o próprio Ghostty intercepta antes. Então a estratégia
escolhida pelo usuário é **manter os atalhos Cmd**, movendo-os pra fora do guard
do Neovide e resolvendo os poucos conflitos com o Ghostty.

Além disso, o usuário quer trazer da config `cpow/neovim-for-newbs` só o que é
genuinamente novo (a config atual já é um superset da do newbs na maior parte):
**oil.nvim**, **alpha-nvim**, **vim-test**, **vim-fugitive** e **catppuccin**.
Também quer **autopairs** (fechar `{`/`(`/`[`/`"` automaticamente) e **aceitar a
sugestão-fantasma do Copilot com Tab**.

**Base:** a config do usuário permanece a base (é mais completa e mais moderna
que a do newbs). Sempre que algo estiver deprecated, usar o equivalente
**moderno/mantido** (a do newbs, na verdade, é mais antiga: `tsserver`→`ts_ls`,
`none-ls`, `lspconfig` estilo antigo — nada disso é copiado).

## Decisões do usuário (já confirmadas)

- Atalhos Cmd: **manter** e configurar o Ghostty pra passá-los ao nvim.
- Plugins novos: **oil.nvim**, **alpha-nvim (dashboard)**, **vim-test**.
- Plugins sobrepostos: **vim-fugitive** e **catppuccin**.
- (Não entram: swagger-preview, nvim-tmux-navigation, none-ls — este último
  descontinuado; o `conform` atual é superior.)

## Conflitos de tecla Cmd (Ghostty ↔ Neovim) e resolução

O Ghostty captura algumas teclas Cmd antes de repassar. Colisões com mapeamentos
`<D-...>` existentes:

| Tecla | Ghostty hoje | nvim quer | Resolução |
|---|---|---|---|
| `Cmd+D` | `new_split:right` | multicursor (`multicursor.lua`) | **liberar no Ghostty** (remover `cmd+d`; splits continuam no `cmd+shift+d`) → Cmd+D vai pro nvim |
| `Cmd+W` | `close_surface` | fechar buffer | **manter no Ghostty**; remover o `<D-w>` morto do nvim e usar `<leader>bd` (já existe) |
| `Cmd+=` `Cmd+-` `Cmd+0` | zoom de fonte | scale do Neovide (só-GUI) | **remover do nvim** (zoom passa a ser do Ghostty, nativo) |

Todos os outros Cmd (`Cmd+P/Shift+P`, `Cmd+Shift+F`, `Cmd+B`, `Cmd+S`, `Cmd+Q`,
`Cmd+Z/Shift+Z`, `Cmd+/`, `Cmd+]`/`[`, `Cmd+1..9`, `Cmd+Shift+G`) não colidem e
passam direto.

## Mudanças

### 1. Remover o Neovide da config

- **`nvim/lua/config/neovide.lua`** — apagar o arquivo.
- **`nvim/init.lua`** — remover a linha `require("config.neovide")` e o comentário
  correspondente.
- **`nvim/lua/config/keymaps.lua`** — trazer pra cá os keymaps genéricos que
  estavam no `neovide.lua` (removendo o guard e as partes só-GUI):
  - Copiar/colar/salvar/sair de macOS: `<D-c>`, `<D-v>`, `<D-s>`, `<D-q>`.
  - Comentar: `<D-/>` (usa `gcc`/`gc` nativo).
  - Busca: `<D-p>`→find_files, `<D-F>`→live_grep, `<D-P>`→commands, `<D-b>`→Neotree.
  - Undo/redo: `<D-z>`, `<D-Z>`.
  - Mover/duplicar linha: `<M-Up>`/`<M-Down>`/`<M-S-Down>`.
  - Indentar: `<D-]>`/`<D-[>`.
  - Abas: loop `<D-1..9>` → `require("bufferline").go_to(i, true)`.
  - **Não** trazer: `guifont`, todos os `vim.g.neovide_*`, `neovide_scale_factor`
    e os keymaps de zoom `<D-=>/<D-->/<D-0>` (morrem; zoom vira função do Ghostty).
  - **Não** trazer `<D-w>` (fica com o Ghostty; buffer fecha via `<leader>bd`).
- **`nvim/lua/config/options.lua`** — manter o bloco de PATH do fnm (é um
  safety-net inofensivo), só ajustar o comentário que fala em Neovide → GUI/PATH.
- **`nvim/lua/plugins/ui.lua`** (linha ~89) — ajustar comentário `neovide .` → `nvim .`.

### 2. Ghostty (`~/.config/ghostty/config`)

- Remover `keybind = cmd+d=new_split:right` (libera Cmd+D pro multicursor do nvim).
  Split à direita continua acessível se o usuário quiser via outro atalho.
- Deixar `cmd+w`, `cmd+=/-/0`, `cmd+t`, `cmd+shift+d` como estão (gestão de
  janela/zoom é responsabilidade do terminal).

### 3. Plugins novos

- **`nvim/lua/plugins/oil.lua`** (novo) — oil.nvim; `-` abre em float
  (`oil.toggle_float`). Convive bem com o neo-tree.
- **`nvim/lua/plugins/alpha.lua`** (novo) — alpha-nvim com tema `startify`
  (dashboard ao abrir `nvim` sem arquivo). Dependência: `nvim-material-icon`
  (o devicons que a config já usa), não o `nvim-web-devicons` do newbs.
  - Ajustar em **`ui.lua`** o autocmd `VimEnter` do neo-tree pra **só** auto-abrir
    quando o argumento é um diretório (`isdirectory(argv(0))`), removendo o caso
    `argc()==0`, pra o alpha ficar dono da tela de abertura vazia.
- **`nvim/lua/plugins/vim-test.lua`** (novo) — vim-test com estratégia
  **`neovim`** (terminal embutido), **não** vimux (vimux exige tmux, que o
  usuário não usa). Keymaps: `<leader>t`… já está ocupado pelo ToggleTerm em
  `keymaps.lua` — usar prefixo livre, ex.: `<leader>tt` (nearest), `<leader>tf`
  (file), `<leader>ta` (suite), `<leader>tl` (last). Sem `preservim/vimux`.
- **`nvim/lua/plugins/autopairs.lua`** (novo) — `windwp/nvim-autopairs`
  (`event = "InsertEnter"`). Fecha `{ ( [ " '` automaticamente. Integra com o
  nvim-cmp: no `completion.lua`, ligar
  `cmp.event:on("confirm_done", require("nvim-autopairs.completion.cmp").on_confirm_done())`
  pra que confirmar uma função no autocomplete já insira os `()`.

### 3b. Aceitar sugestão do Copilot com Tab (sem quebrar o cmp)

Hoje `<Tab>`/`<S-Tab>` estão mapeados no **cmp** (`completion.lua` linhas 32-33)
pra navegar o menu, e o `copilot.lua` (linhas 15-22) evita o Tab de propósito.
Nova lógica (estilo "supertab"):

- **`nvim/lua/plugins/copilot.lua`** — trocar `keymap.accept = "<C-l>"` por
  `accept = false` (o cmp passa a controlar o Tab). Manter `next = "<C-]>"`,
  `prev = "<C-[>"`, `dismiss = "<C-e>"`. Atualizar o comentário.
- **`nvim/lua/plugins/completion.lua`** — trocar `<Tab>`/`<S-Tab>` por uma função
  com prioridade: **1º** se `require("copilot.suggestion").is_visible()` → aceita
  o Copilot; **2º** se `cmp.visible()` → `select_next_item`; **3º** se luasnip
  pode saltar → salta; **senão** `fallback()`. Adicionar `<C-n>`/`<C-p>` pra
  navegar o menu do cmp explicitamente (já que o Tab passa a priorizar o
  Copilot). `<CR>` continua confirmando o cmp; `<S-Tab>` navega/salta pra trás.

### 3c. Cheat sheet no Vim (cheatsheet.nvim) + which-key

Cheat sheet **completa e pesquisável dentro do Neovim** (não só atalhos):

- **`nvim/lua/plugins/cheatsheet.lua`** (novo) — `sudormrfbin/cheatsheet.nvim`,
  `dependencies = { "nvim-telescope/telescope.nvim", "nvim-lua/plenary.nvim" }`.
  Config: `show_help = true`, e apontar pra um cheatsheet do usuário. Keymap
  `<leader>?` → `:Cheatsheet<CR>` (abre o popup Telescope buscável). Desligar as
  seções default que não interessam se poluir (`bundled_cheatsheets`,
  `bundled_plugin_cheatsheets` — deixar só o do usuário + nvim se quiser).
- **`nvim/cheatsheet.txt`** (novo) — cheatsheet **personalizada** lida pelo plugin.
  Formato: `## <seção>` e linhas `descrição | palavras-chave` (`|` é o separador
  do plugin). Cobrir tudo, não só atalhos:
  - **Arquivos/projeto**: `<leader>ff`, `<leader>fg`, `<leader>fr`, `Cmd+P`,
    `Cmd+Shift+F`, `Cmd+B`, `<leader>e`, oil (`-`).
  - **Abas/buffers**: `Cmd+1..9`, `Shift+L/H`, `<leader>bd`.
  - **Edição**: `Cmd+/`, `Cmd+Z/Shift+Z`, `Cmd+]`/`[`, mover/duplicar linha,
    multicursor `Cmd+D`, autopairs.
  - **LSP**: `gd`, `gr`, `K`, `<leader>rn`, `<leader>ca`, `]d`/`[d`, `<leader>xx`.
  - **Copilot**: `Tab` (aceitar), `Ctrl+]`/`Ctrl+[`, `Ctrl+E`.
  - **Git**: `<leader>gg`/`Cmd+Shift+G` (neogit), `:Git` (fugitive), gitsigns.
  - **Testes**: `<leader>tt/tf/ta/tl` (vim-test).
  - **Terminal**: `<leader>t`/ToggleTerm, `<Esc>` sair do modo terminal.
  - **Comandos/fluxos**: `:Copilot auth`, `:Lazy sync`, `:Mason`, `:checkhealth`,
    trocar de tema, como atualizar o backup (rsync).
- **`nvim/lua/plugins/ui.lua`** (which-key) — dar `desc` a todos os keymaps
  novos/movidos pra aparecerem no popup do `<leader>`; opcionalmente registrar
  rótulos de grupo (`<leader>f`=Buscar, `<leader>g`=Git, `<leader>t`=Testes,
  `<leader>x`=Diagnósticos) via `require("which-key").add({...})`.
- **`Atalhos — Neovim_Neovide.pdf`** — **remover** (desatualizado, é do Neovide).
  Substituído pela cheatsheet.nvim + which-key ao vivo + tabelas do README.

## Verificação (end-to-end)

1. **Abrir no Ghostty** (não no Neovide): `nvim` numa pasta de projeto.
2. **Cmd chega ao nvim**: dentro do nvim rodar `:map <D-p>` (deve listar o
   find_files). Apertar **Cmd+P** → abre Telescope; **Cmd+/** comenta; **Cmd+B**
   alterna neo-tree; **Cmd+1/2** troca abas; **Cmd+D** entra em multicursor.
3. **alpha**: abrir `nvim` sem argumento → aparece o dashboard (sem o neo-tree
   brigando por foco). Abrir `nvim <dir>` → neo-tree auto-abre normalmente.
4. **oil**: apertar `-` → abre o painel de edição de arquivos em float.
5. **vim-test**: num arquivo de teste, `<leader>tt` roda o teste mais próximo num
   terminal embutido.
5b. **autopairs**: digitar `{`, `(`, `[`, `"` insere o fechamento automaticamente;
   confirmar uma função no autocomplete já adiciona `()`.
5c. **Copilot com Tab**: com uma sugestão-fantasma visível, **Tab** aceita ela;
   com o menu do autocomplete aberto (sem ghost text), Tab navega o menu e
   `<C-n>`/`<C-p>` também; `<CR>` confirma o item do cmp.
6. **fugitive**: `:Git` mostra o status; neogit (`<leader>gg`/`Cmd+Shift+G`)
   continua funcionando.
7. **Plugins sincronizam sem erro**: `:Lazy sync` e `:checkhealth` sem erros
   novos; `:Lazy` não lista o `neovide.lua` como fonte quebrada.
8. **Sem resíduo de Neovide**: `grep -rin neovide nvim/ install.sh README.md`
   não retorna nada relevante.
